# SupportOps Team Scorecard

Use this optional scorecard to make the activities more engaging.

## Scoring

| Category | Points |
|---|---:|
| App builds and runs | 10 |
| `GET /tickets` works | 10 |
| `POST /tickets` works with validation | 15 |
| SLA calculation works | 15 |
| At least two useful tests or manual checks | 15 |
| Custom instructions improve Copilot output | 10 |
| Reusable prompt file used successfully | 10 |
| Agent delegated feature reviewed by humans | 15 |
| Tenant isolation issue found or prevented | 20 |
| Security fix includes a test | 15 |
| Best prompt shared with the room | 10 |
| Best explanation of Copilot mistake | 10 |
| Cleanest 3-minute demo | 15 |

## Bonus Points

| Bonus | Points |
|---|---:|
| Smallest clean diff for a working feature | 10 |
| Copilot Code Review comment addressed | 10 |
| Team caught a hallucinated file, API, or package | 10 |
| Clear release summary produced | 10 |

## Demo Rubric

Each team gets 3 minutes:

1. What did you build?
2. Which Copilot workflow helped most?
3. What did Copilot get wrong?
4. What evidence proves the app works?
5. What would you improve next?
